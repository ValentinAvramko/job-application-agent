# План рефакторинга ingest_vacancy

Последнее обновление: 2026-04-21

Этот план поддерживается в актуальном состоянии по мере работы. После завершения каждого шага его статус должен обновляться сразу, а результат фиксироваться commit + push.

## Цели

- Упростить `ingest_vacancy.py` и убрать из него низкоуровневые детали интеграций.
- Разделить справочные данные, бизнес-правила и парсинговые эвристики.
- Подготовить ingest к дальнейшему развитию без наращивания монолита.
- Добавить понятный, поэтапный трек внедрения Playwright fallback для JS-heavy сайтов.

## Шаги

1. `completed` Вынести работу с `response-monitoring.xlsx` и обновление `excel_row` из `ingest_vacancy.py` в отдельный модуль интеграции.
2. `completed` Сделать единый каталог стран с каноническим кодом страны, локализованными именами и алиасами.
3. `completed` Вынести `source_channel`, нормализацию способа отклика для `response-monitoring.xlsx` и другие справочные маппинги в data-driven слой.
4. `in_progress` Разделить `ingest_vacancy.py` на provider, normalization и rendering слои.
5. `pending` Добавить Playwright fallback для JS-rendered career sites, когда обычный HTML-fetch не даёт полноценного результата.
6. `pending` Провести финальную зачистку исключений, placeholder policy и тестовой структуры.

## Принципы

- В JSON и другие data-файлы выносятся справочники и конфигурационные соответствия.
- В коде остаются алгоритмы, эвристики, regex-паттерны и правила принятия решений.
- Workflow должен оперировать доменными структурами, а не деталями конкретных форматов хранения.
